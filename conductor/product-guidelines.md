# Product Guidelines

## Brand Messaging & Tone
- **Developer-Friendly & Approachable:** Documentation and API design should prioritize ease of integration and clarity. Use clear examples and approachable language without sacrificing technical accuracy.

## Technical Principles
- **Security First:** Prioritize secure defaults and robust cryptographic practices in every component. Security is never optional.
- **Test-Driven Development (TDD):** Maintain high reliability and maintainability through comprehensive automated testing for all features and bug fixes.
- **Strict Type Safety:** Leverage Dart's strong type system and null safety to catch errors early and provide a better developer experience through self-documenting code.

## Visual Identity
- **Minimalist Documentation:** As this is a support library, the \"visual\" identity is centered on clean, well-structured, and highly readable code and documentation.
- **CLI Design:** Any administrative CLI should follow standard Unix-style conventions (e.g., clear flag descriptions, helpful error messages, and consistent exit codes).
